🎯 Benchmarking

  • Human baseline: Expert answers scored as gold standard
  • Regression tests: 500 representative queries run nightly
  • Adversarial sets: Jailbreak attempts, edge cases, ambiguity
  • Cross-model comparison: Same prompt, different LLMs
  • A/B framework: Prompt v2.1 vs v2.2 with statistical significance

  • External red team: Big-4 cyber + AI ethics firms, annually

  • Bias testing: Demographic parity across HR and customer agents

Decision Framework

The Orchestration Decision: Who Coordinates the Agents?

When a workflow involves multiple steps, multiple agents, or human approval, something must coordinate them. The choice of orchestrator determines flexibility, observability, and long-term maintainability.

Three Orchestration Patterns

🌐 Platform-Native Orchestration

Use Salesforce Flow, SAP BTP Process Automation, ServiceNow Flow Designer, or Power Automate.

  • Best for: Linear workflows inside one platform
  • Pros: Fast, citizen-developer friendly, native UI
  • Cons: Cannot orchestrate cross-platform agents; limited error handling; vendor lock-in
Use when: 1 system, simple flow, no AI reasoning

🛠 Enterprise Gateway Orchestration

The Agent Gateway handles routing, sequencing, and state management across multiple vendor agents.

  • Best for: Cross-platform workflows with 2-4 steps
  • Pros: Centralized observability, unified auth, cost tracking
  • Cons: Not designed for complex multi-turn reasoning
Use when: 2+ systems, defined steps, need governance

🤖 Custom Orchestrator

Build on LangGraph, AutoGen, or a custom state machine for complex multi-agent reasoning with human-in-the-loop.

  • Best for: Complex reasoning, planning, multi-agent debate
  • Pros: Full control, advanced patterns (reflection, ReAct), portable
  • Cons: Requires specialized MLOps talent, longer build cycle
Use when: Strategic IP, complex logic, human approval gates

Orchestrator Decision Flow

Enterprise Default Policy

For any workflow touching customer PII, financial data, or supply chain decisions: Enterprise Gateway minimum. For strategic competitive advantage: Custom Orchestrator. Platform-native orchestration is allowed only for internal productivity tools with no sensitive data.

Citizen Developer Governance

Business users will build Copilot agents and Power Automate flows. Without governance, this becomes shadow AI 2.0. We will enable with guardrails.

Policy: Citizen-built agents must register in the gateway. Unregistered agents are blocked at the network layer. Approved templates only.

🔍

Shadow AI Detection & Redirect

Employees already use ChatGPT, Claude, and Perplexity with company data. We detect this at the network and endpoint level and redirect to the governed platform.

Tactic: Block public AI APIs at corporate firewall. Offer better internal alternatives. Monitor via DLP and endpoint agents.

⚡

Performance & Latency Requirements

Real-time customer-facing agents need sub-second latency. Batch analytics agents can tolerate minutes. We will colocate models and data regionally.

SLA: Customer-facing: <500ms P95. Internal tools: <3s P95. Batch: <1hr. Models cached at edge where needed.

📜 Regulatory Per Jurisdiction

EU requires GDPR compliance + EU AI Act risk classification. China requires data residency + algorithm filing. US has sectoral rules (FDA, SEC). The platform enforces jurisdiction-aware routing automatically.

Year 1: High per-token costs (public LLMs). Year 2: Cost optimization via model routing, caching, and smaller fine-tuned models. Year 3: Predominantly smaller enterprise models, 60-70% cost reduction per inference vs Year 1.

Not all data can go to an LLM. We classify every data source and enforce handling rules automatically through the platform. This is the difference between governed AI and shadow AI.

Restricted / Regulated Data

🔐 Enforcement Mechanisms

Unity Catalog Tags

Every table, file, and document is tagged at ingestion. The GenAI Platform reads tags before retrieval.

Vector Store Filtering

User's Entra ID group membership determines which vector indexes they can query. Restricted chunks are invisible.

Prompt Injection Scan

Before any LLM call, a secondary model scans for attempts to extract restricted data via social engineering.

Output Classification

Every LLM response is classified post-generation. If it contains restricted fragments, it is blocked and alerted.

⚠ Special Handling Rules

  • EU personal data: Processed only in EU regions. No cross-border inference.
  • China data: Azure China 21Vianet only. Separate LLM endpoints.
  • Third-party data: Customer data under NDA requires explicit opt-in per contract.
  • Merger data: Ring-fenced during M&A. No AI access until legal clearance.
  • Auto-downgrade: If classification engine is uncertain, treat as Restricted.

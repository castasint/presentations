The Three Layers at a Glance

Landing Zone

Bronze Layer

Purpose: Ingest everything. Lose nothing.

  • ● Immutable append-only storage
  • ● No schema enforcement (schema-on-read)
  • ● Matches source format: JSON, Avro, CSV
  • ● Partitioned by _ingestion_date
  • ● Contains duplicates, nulls, bad records
💾 Storage: Cheapest object storage (S3, ADLS, GCS)
🥉

spark.readStream \
  .format("cloudFiles") \
  .option("cloudFiles.format", "json") \
  .load("/mnt/raw/events/") \
  .writeStream \
  .option("mergeSchema", "true") \
  .partitionBy("_ingestion_date") \
  .format("delta") \
  .save("/mnt/bronze/events/")
Bronze Pattern

Handling Schema Drift

🛡️ The Problem

Source systems change schemas without warning. A rigid pipeline breaks.

✅ The Solution

Use _rescued_data to catch malformed fields, mergeSchema to evolve automatically.

spark.readStream \
  .format("cloudFiles") \
  .option("cloudFiles.format", "json") \
  .option("cloudFiles.schemaEvolutionMode",
          "rescue") \
  .option("rescuedDataColumn",
          "_rescued_data") \
  .load(source_path)
🏆 Golden Rule: Never reject data at Bronze. Capture everything.

Pattern

Deduplication with Window Functions

from pyspark.sql import Window
from pyspark.sql.functions import row_number

window_spec = Window \
  .partitionBy("customer_id") \
  .orderBy(col("updated_at").desc())

silver_df = bronze_df \
  .filter(col("_rescued_data").isNull()) \
  .withColumn("rn",
    row_number().over(window_spec)) \
  .filter(col("rn") == 1) \
  .drop("rn", "_rescued_data")

🔍 Step 1: Filter

Remove records with _rescued_data (schema violations)

📊 Step 2: Rank

Assign row_number() per customer_id, newest first

🎯 Step 3: Select

Keep only rn = 1 — the latest valid record

Advanced Pattern

Slowly Changing Dimensions Type 2

delta_table.alias("target").merge(
    silver_df.alias("source"),
    """target.customer_id = source.customer_id
       AND target.is_current = true"""
).whenMatchedUpdate(set={
    "is_current": "false",
    "valid_to": "source.updated_at"
}).whenNotMatchedInsert(values={
    "customer_id": "source.customer_id",
    "name": "source.name",
    "email": "source.email",
    "is_current": "true",
    "valid_from": "source.updated_at",
    "valid_to": "null"
}).execute()

Security

Row-Level Security

Users only see data they're authorized to see — enforced transparently by Unity Catalog.

✅ Result

A US manager running SELECT * FROM daily_revenue sees only region = 'US' rows. An admin sees everything.

-- Create row-filter function
CREATE FUNCTION prod.gold.region_filter(
  region STRING
) RETURN BOOLEAN
RETURN CASE
  WHEN is_member('admins') THEN TRUE
  WHEN is_member('us_managers')
    AND region = 'US' THEN TRUE
  WHEN is_member('eu_managers')
    AND region = 'EU' THEN TRUE
  ELSE FALSE
END;

-- Apply to gold table
ALTER TABLE prod.gold.daily_revenue
SET ROW FILTER prod.gold.region_filter
ON (region);
-- Create masking function
CREATE FUNCTION prod.gold.mask_email(
  email STRING
) RETURN STRING
RETURN CASE
  WHEN is_member('pii_access')
    THEN email
  ELSE CONCAT(
    '***@',
    SPLIT(email, '@')[1]
  )
END;

-- Apply to sensitive column
ALTER TABLE prod.silver.dim_customer
ALTER COLUMN email
SET MASK prod.gold.mask_email;

Orchestration

Delta Live Tables + Unity Catalog

import dlt
from pyspark.sql.functions import *

@dlt.table(
  name="bronze.events",
  catalog="prod",
  schema="bronze"
)
def bronze_events():
  return spark.readStream \
    .format("cloudFiles") \
    .load("/Volumes/landing/")

@dlt.expect_or_drop("valid_id",
  "event_id IS NOT NULL")
@dlt.expect_or_drop("valid_ts",
  "event_time IS NOT NULL")
@dlt.table(
  name="silver.events",
  catalog="prod",
  schema="silver"
)
def silver_events():
  return dlt.read("bronze.events") \
    .dropDuplicates(["event_id"])

Watch Out

Anti-Patterns to Avoid

❌ Direct Bronze → Gold

No data quality gate. No reusability. Technical debt accumulates fast.

✅ Fix: Always have Silver

❌ Mutating Bronze Data

Destroys audit trail. Breaks reproducibility. Makes backfills impossible.

✅ Fix: Append-only at Bronze

❌ Over-Partitioning

Millions of tiny files kill query performance. The small file problem.

✅ Fix: Partition by date only

❌ Hardcoded Credentials

Security nightmare. Rotation pain. Compliance violations.

✅ Fix: UC storage credentials

Audit

Compliance & Auditing

Unity Catalog captures every action. Query the audit log for compliance.

What You Can Audit

  • Who created or dropped a table
  • Which queries accessed PII data
  • When permissions were changed
  • Cross-workspace access patterns
-- Audit log queries
SELECT
  user_name,
  action_name,
  request_params.table_name,
  event_time
FROM system.access.audit
WHERE action_name IN (
    'createTable',
    'deleteTable',
    'commandSubmit'
  )
  AND event_date >=
    CURRENT_DATE() - 7
ORDER BY event_time DESC;

-- Table access history
SELECT
  user_name,
  table_name,
  operation,
  query_start_time
FROM system.access.table_lineage
WHERE table_name =
  'prod.gold.daily_revenue';

Core Mechanism

Self-Attention: The Heart of the Transformer

The Intuition

For each token, compute a weighted sum of all other tokens in the sequence. The weights determine "how much to attend to each other token."

QQuery: What am I looking for?
KKey: What do I contain?
VValue: What information do I provide?

Each token creates three vectors by projecting its embedding through learned weight matrices W^Q, W^K, W^V.

Scaled Dot-Product Attention

Attention(Q, K, V) = softmax(QKT / sqrt(d_k)) * V
1Compute similarity scores: QK^T (nxn matrix)
2Scale by sqrt(d_k) to prevent softmax saturation
3Apply softmax to get attention weights (row-wise)
4Weighted sum with V to get output
Why scale by sqrt(d_k)? For large d_k, dot products grow in magnitude, pushing softmax into regions with extremely small gradients. Scaling stabilizes training.

Key Properties

YGlobal receptive field: Every token sees every other token in one layer
YParallelizable: All attention computations happen simultaneously
YPath length O(1): Any two tokens are directly connected
!Quadratic complexity: O(n^2) in sequence length
Complexity: O(n^2 * d) time and O(n^2) memory. This is the fundamental bottleneck for long sequences.

Position Information

Positional Encoding

The Problem

Self-attention is permutation-invariant. Without positional information, the model cannot distinguish between "The cat sat" and "sat cat The".

Original Solution: Sinusoidal

PE(pos, 2i) = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d)

These functions create unique, bounded encodings with a linear relationship between relative positions.

Modern Variants

ALearned Positional Embeddings: Simple lookup table, trained end-to-end. Used in BERT, GPT.
BRoPE (Rotary): Rotates query/key vectors by position-dependent angles. Used in LLaMA, PaLM.
CALiBi: Adds linear biases to attention scores instead of embeddings. Better extrapolation.
DRelative Positional Encodings: Encode distance between tokens, not absolute position.
RoPE is now dominant in LLMs because it naturally generalizes to longer sequences than seen during training.
Building Blocks

FFN, Residuals & LayerNorm

FFN

Feed-Forward Network

FFN(x) = max(0, xW1 + b1)W2 + b2
  • Applied position-wise
  • Inner dim: 4x model dim (d_ff = 2048)
  • ~2/3 of total parameters
  • Modern: GELU or SwiGLU activation
Res

Residual Connections

output = x + Sublayer(x)
  • Enable gradient flow in deep stacks
  • Transformer base: 6 layers
  • Modern LLMs: 96+ layers
  • Without residuals: training fails
LN

Layer Normalization

LN(x) = gamma * (x - mu) / sqrt(sigma^2 + eps) + beta
  • Normalizes across feature dimension
  • Stabilizes training
  • Pre-Norm (modern): LN first
  • Post-Norm (original): LN after
Pre-Norm vs Post-Norm: Pre-Norm (LN -> Sublayer -> Add) is now standard because it enables training much deeper networks without instability.

Vision Transformer (ViT)

Applying Transformers to Computer Vision

"An Image is Worth 16x16 Words" - Dosovitskiy et al., ICLR 2021

Core Idea

Treat Images as Sequences

The Insight

What if we chop an image into fixed-size patches, flatten them, and treat each patch as a "token" in a sequence? Then we can apply a standard Transformer encoder!

1Split image into patches (e.g., 16x16)
2Flatten each patch to a vector
3Embed via linear projection + position
4Feed into standard Transformer encoder
5Classify using a special [CLS] token

Image -> Patch Sequence

224x224 image -> 14x14 = 196 patches of 16x16 pixels each

III

Mixture of Experts (MoE)

Conditional Computation for Massive Scale

Scale parameters without scaling compute. Activate only what you need.

Core Idea

Why MoE? The Scaling Problem

Dense Models Hit Limits

!Compute cost: Every parameter is used for every input
!Training time: Proportional to total parameters x data
!Inference cost: Full model must be loaded and run
!Memory wall: Hardware cannot keep up with model size

GPT-4 is rumored to be ~1.8T parameters. Running all of them for every token is incredibly expensive.

MoE Solution: Conditional Compute

YSparsely activate parameters per input
YScale total params without scaling FLOPs
YDifferent "experts" learn different specializations
YGating network routes each token to top-k experts
Key insight: A 1T parameter MoE with top-2 routing uses roughly the same FLOPs as a 100B dense model - but with 10x the capacity.
Architecture

MoE Layer: How It Works

Sparsely-Gated MoE Layer

Input Token x
↓
Gating Network
↓ softmax -> top-k
Expert 1
Expert 2
Expert 3
...
Expert N
↓ weighted sum
Output y

Only top-k experts are activated per token

Gating Function

G(x) = TopK(softmax(W_g * x), k)

The gating network outputs a probability distribution over N experts. We keep only the top-k values and set the rest to 0.

Load Balancing

Without constraints, the gating network may route everything to a few "popular" experts.

AAuxiliary loss: Encourage uniform expert usage
BCapacity factor: Limit tokens per expert
CExpert choice: Tokens pick experts, not vice versa
Mathematics

MoE: Formulas & Real-World Models

MoE Layer Output

y = x + Sumi in TopK G(x)i * Ei(x)

Where E_i is the i-th expert (typically an FFN). The residual connection is preserved.

Auxiliary Load Balancing Loss

Laux = alpha * N * Sumi=1N fi * Pi

f_i = fraction of tokens routed to expert i. P_i = mean routing probability to expert i. Minimized when both are 1/N.

Trade-offs

MoE Challenges & Solutions

Challenges

!Training instability: Gating can collapse to few experts
!Communication overhead: All-to-all token routing
!Memory fragmentation: Each device holds some experts
!Load imbalance: Some experts overflow, others idle
!Fine-tuning difficulty: Overfitting on small datasets

Solutions & Mitigations

ANoisy Top-K Gating: Add noise before softmax for exploration
BExpert Choice Routing: Experts choose tokens, not vice versa
CEP (Expert Parallelism): Distribute experts across devices
DCapacity factor: Drop tokens beyond expert capacity
ERouter z-loss: Penalize extreme gating logits
Mixtral's breakthrough: By using only 8 experts (not 1000s), Mixtral made MoE practical for consumer GPUs. It matches 70B dense models with ~4x faster inference.

LoRA & Parameter-Efficient Fine-Tuning

Fine-Tune Giants Without Breaking the Bank

Train 0.1% of parameters. Achieve 99% of full fine-tune quality.

Motivation

The Fine-Tuning Problem

Full Fine-Tuning Is Expensive

!LLaMA-3 70B: 140GB in FP16 just to load
!Optimizer states: Adam needs 2x params (momentum + variance)
!Gradients: Another 140GB for backward pass
!Storage: Each task needs a full model copy
Total memory for full fine-tuning LLaMA-3 70B with Adam: ~420GB+ - requiring multiple A100 GPUs.

PEFT to the Rescue

YFewer trainable params: 0.1% - 1% of total
YLess memory: No optimizer state for frozen weights
YFaster training: Less backward computation
YStorage efficient: Small adapters per task
YNo inference latency: Merge adapters into base model
LoRA

LoRA: Low-Rank Adaptation

Hu et al. (2021). LoRA: Low-Rank Adaptation of Large Language Models. ICLR.

The Core Hypothesis

The weight updates during fine-tuning have a low "intrinsic rank". Instead of updating the full matrix W in R^(d x k), we learn a low-rank decomposition.

W' = W0 + delta W = W0 + BA

W_0 is frozen. B in R^(d x r), A in R^(r x k) are trainable. Rank r much less than min(d,k).

Why It Works

1Over-parameterization: Pretrained models live in a low-dim manifold
2A is initialized with random Gaussian
3B is initialized to zero -> delta W = 0 at start
4Scaling: h = W_0 x + (alpha/r) * BAx
Alpha is a tunable hyperparameter. Typically alpha = 2r. At inference, W' can be merged back into W_0 with zero overhead.

Practical Guide

LoRA: Best Practices

Where to Apply LoRA

YQ and V projections: Minimal config, good results
YQ, K, V, O: Standard config (attention only)
YAll linear layers: Best quality, more params
!Embeddings: Usually not needed
!LayerNorm: Sometimes fine-tuned (not LoRA'd)

Hyperparameters

rRank: Start with 8-16. Increase for complex tasks.
aAlpha: Typically alpha = 2r. Controls adaptation strength.
lrLearning rate: 1e-4 to 1e-3 (higher than full fine-tune)
DODropout: 0.05-0.1 on LoRA layers only
TTarget modules: q_proj, v_proj minimum

AdaLoRA

Dynamically allocates rank budget across layers based on importance. More efficient than fixed-rank LoRA.

Zhang et al. (2023). AdaLoRA: Adaptive Budget Allocation for Parameter-Efficient Fine-Tuning. ICLR.

DoRA

Weight-Decomposed LoRA. Decomposes pre-trained weights into magnitude and direction, only applying LoRA to direction.

Liu et al. (2024). DoRA: Weight-Decomposed Low-Rank Adaptation.

V

QLoRA: Quantized LoRA

Fine-Tune 65B Models on a Single 48GB GPU

4-bit quantization + double quantization + paged optimizers = extreme efficiency

Foundation

Quantization: Storing Weights in Fewer Bits

FP16 (Standard)

16
bits per parameter
  • 5 bits exponent
  • 10 bits mantissa
  • 1 sign bit
  • Dynamic range: ~10^(+-5)

INT8 Quantization

8
bits per parameter
  • Linear mapping: w_q = round(w / s + z)
  • Requires scale (s) and zero-point (z)
  • 2x memory reduction
  • Minimal accuracy loss

NF4 (QLoRA)

4
bits per parameter
  • Normal Float 4: Optimal for normally-distributed weights
  • Quantile-based quantization
  • 4x memory reduction vs FP16
  • Block-wise (64) for stability
Normal Float (NF): Instead of uniformly spaced values, NF uses quantiles of a normal distribution. Since neural network weights are approximately normal, this minimizes quantization error.
QLoRA

QLoRA: The Three Innovations

Dettmers et al. (2023). QLoRA: Efficient Finetuning of Quantized LLMs. NeurIPS.

4b

4-bit Normal Float

Base model weights stored in NF4 (4-bit). During forward pass, dequantize on-the-fly to BF16 for computation.

Wdequant = dequant(WNF4)
DQ

Double Quantization

Quantize the quantization constants themselves! Block constants (32-bit) are quantized to 8-bit. Extra 0.4 bits/param saved.

c2FP8 = quant(c1FP32)
Pg

Paged Optimizers

Use NVIDIA unified memory to page optimizer states to CPU RAM when GPU OOM. Automatic page fault handling.

GPU <-> CPU via cudaMallocManaged

Implementation

QLoRA in Practice (Hugging Face)

from transformers import (
    AutoModelForCausalLM,
    AutoTokenizer,
    BitsAndBytesConfig
)
from peft import (
    LoraConfig,
    get_peft_model,
    prepare_model_for_kbit_training
)

# 1. Configure 4-bit quantization
bnb_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16,
    bnb_4bit_use_double_quant=True,
)

# 2. Load model in 4-bit
model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-2-70b",
    quantization_config=bnb_config,
    device_map="auto",
)

# 3. Prepare for training
model = prepare_model_for_kbit_training(model)

# 4. Configure LoRA
lora_config = LoraConfig(
    r=16,
    lora_alpha=32,
    target_modules=["q_proj",
                    "v_proj",
                    "k_proj",
                    "o_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type="CAUSAL_LM",
)

# 5. Apply LoRA to quantized model
model = get_peft_model(model, lora_config)

# 6. Train!
# Only LoRA params are trainable
model.print_trainable_parameters()
# Output: trainable params: 41M
#         all params: 68.9B
#         % trainable: 0.06%

# 7. Merge for inference (zero overhead)
model = model.merge_and_unload()
Landscape

Other PEFT Methods

Prefix Tuning

Prepend trainable "prefix" vectors to keys and values in every layer.

[P_x, K, V]

Li & Liang (2021). 0.1% params. Good for generation.

P-Tuning v2

Continuous prompts at every layer, not just input. Much better than v1.

h_i = f(h_(i-1), p_i)

Liu et al. (2022). NLU tasks. 0.1-0.5% params.

Adapters

Insert small bottleneck FFNs after attention and FFN layers.

h -> h + W_up(ReLU(W_down h))

Houlsby et al. (2019). ~3% params. Task composition.

Prompt Tuning

Learn soft prompt tokens prepended to input. Only embeddings trainable.

[p_1, ..., p_t, x]

Lester et al. (2021). ~0.01% params. Needs scale.

IA3

Learn element-wise scaling vectors for keys, values, and FFN activations.

k -> l_k * k

Liu et al. (2022). Similar params to LoRA. Often better.

(Q)LoRA

Low-rank decomposition of weight updates. The current gold standard.

W' = W + BA

Hu et al. (2021). Most popular. Best quality/params tradeoff.

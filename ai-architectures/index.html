Foundations of Deep Learning

From Perceptrons to Deep Neural Networks

The Perceptron (1958)

  • Frank Rosenblatt's foundational model
  • Binary classifier based on linear threshold
  • y = σ(w·x + b) where σ is step function
  • Learned via error-correction rule
  • Could only solve linearly separable problems

Rosenblatt, F. (1958). The Perceptron: A Probabilistic Model for Information Storage and Organization in the Brain

output = { 1 if w·x + b > 0
          { 0 otherwise
Limitation: Minsky & Papert (1969) proved perceptrons cannot learn XOR — a non-linearly separable function

Multi-Layer Perceptron (MLP)

  • Stacked layers of perceptrons
  • Hidden layers enable non-linear representation
  • Universal Approximation Theorem
  • Requires non-linear activation functions
  • Trained via backpropagation (Rumelhart et al., 1986)

Backpropagation & Gradient Descent

Forward Pass

  • Compute predictions through layers
  • z[l] = W[l]a[l-1] + b[l]
  • a[l] = g(z[l])

Backward Pass

  • Compute gradients via chain rule
  • ∂L/∂W[l] = δ[l] · a[l-1]T
  • Update: W := W - α·∂L/∂W

Optimizers

  • SGD: Simple, needs tuning
  • Momentum: Accelerates in consistent directions
  • AdaGrad: Per-parameter learning rates
  • RMSprop: Moving average of squared gradients
  • Adam: Momentum + RMSprop (most popular)
  • AdamW: Decoupled weight decay
  • Lion: Memory-efficient, sign-based

LSTM & GRU: Gated Recurrent Units

LSTM (1997)

  • Cell state + three gates
  • Forget gate: ft = σ(Wf·[ht-1, xt])
  • Input gate: it = σ(Wi·[ht-1, xt])
  • Output gate: ot = σ(Wo·[ht-1, xt])
  • Solves vanishing gradients

Hochreiter & Schmidhuber (1997)

GRU (2014)

  • Simplified gating mechanism
  • Update gate: zt
  • Reset gate: rt
  • No separate cell state
  • Fewer parameters, faster training
  • Often comparable performance to LSTM

Attention Mechanism (Bahdanau et al., 2015)

  • Addresses the bottleneck problem
  • Decoder attends to all encoder hidden states
  • Attention scores: eij = score(st-1, hj)
  • Weights: αij = softmax(eij)
  • Context: ct = Σ αijhj
Score Functions:
• Dot Product: score(s,h) = sTh
• Concat: score(s,h) = vTtanh(Wss + Whh)
• General: score(s,h) = sTWah
• Location-based: learns alignment directly

Bahdanau et al. (2015). Neural Machine Translation by Jointly Learning to Align and Translate

Scaled Dot-Product Attention

Self-Attention: The Core Mechanism

  • For each token, compute attention with all other tokens
  • Three learned projections per token:
  • • Query (Q): What am I looking for?
  • • Key (K): What do I contain?
  • • Value (V): What information do I provide?
Attention(Q, K, V) = softmax(QKT / √dk) · V
Scaling factor √dk: Prevents dot products from growing too large in magnitude, pushing softmax into regions with small gradients.

Self-Attention: Step by Step

Input: X ∈ ℝn×d (n tokens, d dimensions)

  1. Project: Q = XWQ, K = XWK, V = XWV where W ∈ ℝd×dk
  2. Compute scores: S = QKT ∈ ℝn×n
  3. Scale: S = S / √dk
  4. Apply softmax row-wise: A = softmax(S) ∈ ℝn×n (attention weights)
  5. Weighted sum: Output = AV ∈ ℝn×dv
Complexity: O(n²·d) time, O(n²) memory for attention matrix. This quadratic complexity is the fundamental bottleneck for long sequences.

Multi-Head Attention

Multi-Head Attention

  • Run h parallel attention heads
  • Each head learns different types of relationships
  • One head: syntactic dependencies
  • Another: semantic relationships
  • Another: long-range dependencies
MultiHead(Q,K,V) = Concat(head₁,...,headₕ)WO

where headi = Attention(QWiQ, KWiK, VWiV)

Typical config: h=8 heads, dk=dv=d/h=64

Positional Encoding

  • Transformers have no inherent notion of position
  • Must inject position information
  • Original: sinusoidal functions
  • Learned embeddings also common
  • RoPE (Rotary) now dominant in LLMs
PE(pos, 2i) = sin(pos / 100002i/d)
PE(pos, 2i+1) = cos(pos / 100002i/d)
Properties:
• Unique encoding for each position
• Bounded values
• Linear relation between relative positions

Layer Normalization & Residual Connections

Layer Normalization

  • Normalizes across feature dimension
  • LN(x) = γ ⊙ (x - μ) / √(σ² + ε) + β
  • Stabilizes training of deep networks
  • Pre-Norm vs Post-Norm variants

Ba et al. (2016). Layer Normalization

Residual Connections

  • output = x + Sublayer(x)
  • Enables gradient flow in deep stacks
  • Transformer base: 6 encoder + 6 decoder layers
  • Modern LLMs: 96+ layers
Pre-Norm (modern): LN → Sublayer → Add
Post-Norm (original): Sublayer → Add → LN

Feed-Forward Network (FFN)

  • Applied position-wise (same for all positions)
  • Two linear transformations with ReLU/GELU
  • FFN(x) = max(0, xW₁ + b₁)W₂ + b₂
  • Inner dimension: 4× model dimension (dff = 4d)
  • Major source of parameters (~2/3 of total)

Cross-Attention (Encoder-Decoder)

  • Decoder queries attend to encoder outputs
  • Q from decoder, K/V from encoder
  • Alignment between source and target
  • Replaces the fixed context vector
  • Dynamic, token-level alignment

Usage: Machine translation, summarization, any seq2seq task. Modern LLMs often omit cross-attention in favor of decoder-only architectures.

BERT, GPT, T5, and Beyond

The Three Transformer Families

Encoder-Only

BERT, RoBERTa, ALBERT, DeBERTa

  • Bidirectional attention
  • See full context
  • Best for: understanding
  • Tasks: classification, NER, QA extraction

Decoder-Only

GPT, LLaMA, PaLM, Claude

  • Causal (masked) attention
  • Left-to-right generation
  • Best for: generation
  • Tasks: text generation, code, chat

Encoder-Decoder

T5, BART, UL2

  • Full encoder + masked decoder
  • Cross-attention
  • Best for: translation, summarization
  • Unified text-to-text framework

RoPE: Rotary Position Embedding

  • Su et al. (2021)
  • Encodes position by rotating query/key vectors
  • f(q,m) = qe^(imθ) in complex space
  • Relative positions naturally encoded
  • Excellent extrapolation properties
  • Standard in modern LLMs

Su et al. (2021). RoFormer: Enhanced Transformer with Rotary Position Embedding

2D case:
R_Theta,m =
[[cos(mθ), -sin(mθ)],
[sin(mθ), cos(mθ)]]

f(q,m) = R_Theta,m * q

ALiBi: Attention with Linear Biases

  • Press et al. (2021)
  • No position embeddings at all
  • Adds linear bias to attention scores:
  • softmax(qk^T + m*[-(i-j)])
  • Each head has learned slope m
  • Strong extrapolation to longer sequences

Press et al. (2021). Train Short, Test Long: Attention with Linear Biases

Advantage: No position embedding lookup. The bias naturally penalizes distant attention, creating an inductive bias for locality.

Scaling Laws & Large Models

The Era of Scale

Scaling Laws (Kaplan et al., 2020)

  • OpenAI empirical study
  • Loss scales as a power law with:
  • • Model parameters (N)
  • • Dataset size (D)
  • • Compute (C)
  • L(N) ~ N^(-α) where α ~ 0.076
  • L(C) ~ C^(-β) where β ~ 0.050

Kaplan et al. (2020). Scaling Laws for Neural Language Models

Key Finding: Performance improves predictably with scale. Larger models are more sample-efficient — they reach the same performance with fewer optimization steps.

Power Law Predictable Emergence

  • Hoffmann et al. (DeepMind)
  • Revised scaling laws
  • Kaplan: fix data, scale params
  • Chinchilla: params and data should scale equally
  • Optimal: D ~ 20N (tokens = 20 x params)
  • GPT-3 was undertrained

Conditional Computation at Scale

Mixture of Experts Routing

What is Mixture of Experts?

  • Idea: Not all parameters need to activate for every input
  • Divide model into "experts" (specialized sub-networks)
  • Router/Gate: Decides which experts to use
  • Only a subset of experts active per token
  • Scale parameters without scaling compute

MoE Formalization

y = Sum(i=1 to E) G(x)_i * E_i(x)
  • E_i(x): Expert i (typically an FFN)
  • G(x)_i: Gate value for expert i
  • Sparsity: Most G(x)_i = 0 (only top-k non-zero)
G(x) = Softmax(TopK(W_g * x, k))

TopK sets all but the k largest values to -infinity before softmax

MoE in Transformers

  • Replace every other FFN with MoE layer
  • Attention layers remain dense
  • Each token routed independently
  • Experts placed on different devices
  • Enables massive parameter counts

Load Balancing in MoE

The Problem

  • Router may collapse to few experts
  • Some experts overloaded, others idle
  • Wastes capacity, hurts performance

Solutions

  • Auxiliary loss: Encourage uniform routing
  • Expert choice: Tokens choose experts, experts choose tokens
  • Load balancing loss: L_load = alpha * N * Sum_i f_i * P_i
Auxiliary Loss:
loss = alpha * E * Sum(i=1 to E) f_i * P_i

f_i = fraction of tokens to expert i
P_i = mean routing probability to expert i
alpha = hyperparameter (typically 0.01)

Expert Choice Routing (Zhou et al., 2022)

  • Reverse the paradigm:
  • Each expert selects top-k tokens
  • Guarantees perfect load balancing
  • No token dropping
  • Some tokens may be selected by multiple experts
  • Others may not be selected at all

Zhou et al. (2022). Mixture-of-Experts with Expert Choice Routing

Comparison:

Token Choice: Tokens -> pick experts (imbalanced)

Expert Choice: Experts -> pick tokens (balanced)

Expert choice achieves better perplexity with guaranteed balance.

Linear Attention & Approximations

  • Problem: O(n^2) attention is expensive
  • Goal: Approximate or replace attention
  • Approaches:
  • • Sparse attention patterns
  • • Kernel methods (Linear Transformer)
  • • Low-rank approximations
  • • State space models
Linear Attention:
softmax(QK^T)V ~ phi(Q)phi(K)^T V

Using feature map phi, reorder:
= phi(Q)(phi(K)^T V)

Complexity: O(n^2 d) -> O(n d^2)

FlashAttention

  • Dao et al. (2022)
  • IO-aware exact attention
  • No approximation — exact same output
  • Tiling to fit in SRAM
  • Fewer HBM reads/writes
  • 2-4x speedup, 10-20x memory savings

FlashAttention-2 (2023)

  • Better parallelism
  • Further 2x speedup

Dao et al. (2022). FlashAttention: Fast and Memory-Efficient Exact Attention

Memory Hierarchy:

HBM (High Bandwidth Memory): Large, slow
SRAM (Shared Memory): Small, fast

FlashAttention splits attention into tiles that fit in SRAM, avoiding round-trips to HBM.

Ring Attention

  • Liu et al. (2023)
  • Extends FlashAttention to infinite context
  • Blockwise computation across devices
  • Key-value blocks circulate in ring topology
  • Computation and communication overlapped
  • Enables millions of tokens

Liu et al. (2023). Ring Attention with Blockwise Transformers

Devices arranged in ring:

Device 0 -> Device 1 -> Device 2 -> ... -> Device 0

Each device computes local attention while passing KV blocks to next device.

State Space Models (S4)

  • Gu et al. (2021)
  • Inspired by continuous state space
  • h'(t) = A h(t) + B x(t)
  • y(t) = C h(t) + D x(t)
  • Discretized for sequences
  • O(n) training and inference
  • Long-range dependencies

Gu et al. (2021). Efficiently Modeling Long Sequences with Structured State Spaces

Continuous:
h_dot(t) = A h(t) + B x(t)
y(t) = C h(t)

Discretized:
h_k = A_bar h_{k-1} + B_bar x_k
y_k = C h_k

Mamba Selective State Space Model

Mamba: Selective State Spaces

  • Gu & Dao (2023)
  • Selection mechanism: B, C, Delta depend on input
  • Hardware-aware parallel scan
  • O(n) complexity like S4
  • Strong empirical results
  • Challenges Transformer dominance

Gu & Dao (2023). Mamba: Linear-Time Sequence Modeling with Selective State Spaces

Key Difference from S4:

S4: B, C, Delta are fixed (input-independent)

Mamba: B, C, Delta = f(x) (input-dependent)

This "selection" allows content-aware reasoning while maintaining linear complexity.

RWKV: RNN with Transformer Benefits

  • Peng (2023)
  • Parallelizable training like Transformer
  • Constant memory inference like RNN
  • No attention mechanism
  • Channel mixing + time decay
  • 14B parameter models trained

Peng (2023). RWKV: Reinventing RNNs for the Transformer Era

RWKV Attention:

a_t = exp(-w) * a_{t-1} + exp(k_t) * v_t
b_t = exp(-w) * b_{t-1} + exp(k_t)
o_t = sigmoid(r_t) * (a_t / b_t)

w = time decay (learned per channel)

RetNet: Retentive Network

  • Sun et al. (2023)
  • Microsoft's "Transformer killer"

z = mu + sigma elementwise epsilon, epsilon ~ N(0,I)

Objective:

min_G max_D V(D,G) =
E_{x~p_data}[log D(x)] +
E_{z~p_z}[log(1-D(G(z)))]

RLHF: Reinforcement Learning from Human Feedback

  • Christiano et al. (2017), Ouyang et al. (2022)
  • 1. Train Reward Model (RM):
  • • Humans rank model outputs
  • • Bradley-Terry model: P(y1>y2) = sigma(r(x,y1)-r(x,y2))
  • 2. Optimize Policy with PPO:
  • • Maximize reward + KL penalty

Ouyang et al. (2022). Training language models to follow instructions with human feedback

PPO Objective:

max_pi E[r(x,y)]
- beta * D_KL(pi||pi_ref)

RM Loss:
-log sigma(r_theta(x,y_w) - r_theta(x,y_l))

DPO: Direct Preference Optimization

DPO: Direct Preference Optimization

  • Rafailov et al. (2023)
  • No reward model needed!
  • Directly optimize from preferences
  • Closed-form solution to RLHF
  • Simpler, more stable than PPO
  • Mathematically equivalent under Bradley-Terry model

Rafailov et al. (2023). Direct Preference Optimization: Your Language Model is Secretly a Reward Model

DPO Loss:

L = -log sigma(beta log[pi(y_w|x)/pi_ref(y_w|x)]
- beta log[pi(y_l|x)/pi_ref(y_l|x)])
Advantage: Single-stage training. No unstable RL loop. Often matches or exceeds PPO quality.

  • Problem: Autoregressive generation recomputes K,V for all past tokens
  • Solution: Cache K,V matrices
  • Reduces complexity from O(n^3) to O(n^2)
  • Memory bottleneck for long contexts
  • Compression methods:
  • • H2O: Evict less important KV pairs
  • • StreamingLLM: Keep sink tokens + window
  • • SnapKV: Select important KV in prefill
Memory Growth:

For layer l, head h:
KV cache = 2 x n x d_head x bytes

LLaMA-70B, 32K context:
~160 GB just for KV cache!

  • Hasani et al. (MIT, 2020)
  • Continuous-time neural networks
  • Neural ODEs with input-dependent dynamics

  • Interpretable with few neurons
  • Truly causal (no fixed window)

Liquid Neuron:

dx/dt = -[1/tau + f(x,I)]x + f(x,I)*sigma(Ax + BI)

tau = time constant
f(x,I) = input-dependent term

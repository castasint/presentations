Design Principle

The AI-Native Philosophy

Why we replaced every traditional component with an LLM

What "AI-Only" Means Here

✨LLM Extracts: GPT-4o Vision reads PDFs directly and outputs structured JSON — no OCR pipeline, no table parsers
✨LLM Reasons: Collusion is detected by asking the model to compare quotes and judge coordination — no Benford, no Z-score
✨LLM Validates: Compliance is checked by having the model read all 7 files and judge completeness — no rule engine
✨LLM Benchmarks: Price fairness is assessed by the model against historical bid memory — no SQL aggregates
✨LLM Narrates: Every finding explained in natural language with citations — no dashboard-only outputs

Why This Approach Wins

▲Handles Any Format: Suppliers can use any table layout, any column names, any language — the LLM adapts instantly
▲No Threshold Tuning: No more "is 95% similarity collusion?" — the model reasons from context, not fixed cutoffs
▲Emergent Detection: LLMs spot patterns never explicitly trained — subtle linguistic cues in emails, formatting similarities across quotes
▲Human-Readable: Every output is a complete explanation, not a number requiring interpretation
▲Single Model Stack: One GPT-4o deployment does extraction, analysis, validation, and reporting — operational simplicity
The Trade-Off Is Acceptable
Statistical methods are deterministic and cheap. LLMs are probabilistic and cost more per document. But for procurement risk — where a single missed collusion can cost millions — the interpretability, adaptability, and comprehensiveness of LLM reasoning outweigh the compute cost. We use deterministic caching and prompt templates to keep costs predictable.

GPT-4o Vision — The Universal Parser

1Image Input: PDF pages converted to images → fed to GPT-4o Vision
2JSON Output: Prompt constrains model to emit valid JSON with item_code, description, qty, unit_price, total
3Multi-Table: Model identifies cover sheet, terms page, and item tables — returns only item tables
4Currency Handling: Model normalizes currencies to standard codes (USD, EUR, CNY) automatically
5Confidence: Each extraction tagged with model confidence; low-confidence items flagged for human review

Schema Resolver — Flexible Mapping

AHeader Classification: LLM reads extracted column names and maps "Amt", "Value", "Cost" → unit_price
BType Inference: LLM detects whether a column is quantity, price, date, or text description
CCustom Fields: Unmapped columns stored as JSONB in Cosmos DB — no data loss, future flexibility
DCross-Supplier Alignment: Ensures L1, L2, L3 quotes use consistent item_codes for comparison

Collusion Agent — How It Thinks

"I see three quotes for Bid 2024-089. Supplier A and B both priced Item X at exactly $12,450. Their delivery terms are identical: '45 days FOB Shanghai.' Both use the same font, same header layout, and both omit warranty clauses that Supplier C included. The probability of independent bids being this similar is extremely low. I rate collusion risk: HIGH."
●Reasons over price vectors, terms text, and formatting cues simultaneously
●No fixed threshold — judges from context and historical patterns
●Chain-of-thought prompt forces model to explain its reasoning step by step

Anomaly Agent — How It Thinks

"Retrieving 18 historical bids for 'server rack 42U.' Prices ranged from $1,200 to $1,850 with a typical cluster around $1,400. Supplier D quoted $2,900 — more than double the historical high. No justification in terms (standard delivery, no special features). I flag this as anomalous and recommend requesting a price breakdown."
●Retrieves relevant historical bids from Cosmos DB via semantic search
●Judges price fairness against remembered market context, not just averages
●Flags both extreme outliers AND suspiciously uniform quotes

Compliance Agent — How It Thinks

"I count 6 files in this folder, not 7. The L2 email PDF is missing. The Excel summary lists a total of $145,000 but the sum of extracted quote lines is $142,300 — a $2,700 discrepancy. The approval email from the procurement manager is dated 2 days after the bid deadline. I flag: MISSING_DOC, SUMMARY_MISMATCH, LATE_APPROVAL."
●Counts documents by type, names, and content — not just file extensions
●Cross-checks Excel totals against extracted PDF line items
●Reads email body text to identify approver names and dates

Narrative Agent — The Synthesizer

"This bid presents a CRITICAL risk profile. The Collusion Agent identified 94% price similarity between Suppliers A and B, with identical delivery terms. The Anomaly Agent found both quotes 40% above historical benchmarks. The Compliance Agent noted a missing L2 email. Combined, these signals strongly suggest coordinated bidding. Recommend immediate investigation before award."
●Aggregates all agent outputs into a single coherent risk story
●Assigns overall severity by weighing multiple signals
●Generates citations linking every claim to source document pages

The Secret Sauce

Prompt Architecture & Engineering

How we make LLMs reliable for procurement risk detection

Extraction Prompt Pattern

You are a procurement document parser. Given the attached PDF page images,
extract all line items into this exact JSON schema:

{
  "items": [{
    "item_code": "string",
    "description": "string",
    "quantity": number,
    "unit_price": number,
    "currency": "USD|EUR|CNY",
    "confidence": "high|medium|low"
  }],
  "supplier_name": "string",
  "total_amount": number
}

Rules:
- If a column header is ambiguous, infer from context.
- If a cell is unreadable, set confidence to "low".
- Do not hallucinate values. If unsure, omit the field.

Collusion Prompt Pattern

You are a forensic procurement analyst. I will give you 3 supplier quotes
for the same bid. Analyze them for signs of collusion or bid rigging.

Consider:
1. Price similarity across line items
2. Identical or near-identical terms and conditions
3. Suspicious formatting similarities (fonts, layout)
4. Unusual patterns (one supplier bids high on all items except one)
5. Missing information that all others include

Output a JSON with:
- "collusion_risk": "none|low|medium|high|critical"
- "reasoning": "detailed paragraph explaining your analysis"
- "evidence": ["specific observation 1", "observation 2"]

Reliability Techniques

1JSON Mode: Forces GPT-4o to emit parseable structured output — no regex extraction from prose
2Few-Shot Examples: Prompt includes 2-3 examples of correct extractions and collusion judgments
3Chain-of-Thought: Model must explain reasoning before giving final score — catches hallucinations
4Self-Consistency: Run same prompt 3× with temperature=0.7; majority vote on final classification

Cost & Latency Controls

APrompt Caching: Static prompt prefixes (schema, examples) cached via Azure OpenAI prompt caching
BBatch Processing: Multiple PDF pages sent in single API call; parallel agent execution
CTiered Model: GPT-4o-mini for extraction (cheaper), GPT-4o for collusion reasoning (higher quality)
DResult Caching: Cosmos DB caches extraction results by file hash — identical files never reprocessed

Security in an AI-Native World

Governance for autonomous LLM agents handling sensitive procurement data

Data Boundary

✓Private Endpoints: Azure OpenAI, Cosmos DB, Blob — no public internet exposure
✓No Training: Your bid data is never used to train foundation models (Azure OpenAI enterprise promise)
✓Content Filtering: Azure OpenAI built-in filters block prompt injection and jailbreaks
✓PII Redaction: Supplier addresses, phones redacted before LLM processing via Presidio

Agent Governance

✨Human-in-the-Loop: CRITICAL severity flags require human approval before any procurement action
✨Confidence Gates: Agent outputs below 0.8 confidence automatically routed to human review queue
✨Prompt Versioning: All prompts versioned in Dataverse — rollback to previous behavior instantly
✨Cost Caps: Daily token budget per agent; auto-shutdown if exceeded

Audit & Explainability

★Full Trace: Every agent decision stored with prompt, response, reasoning chain, and cost
★Reproducible: Same prompt + same document hash = deterministic re-run for audit verification
★Immutable Logs: Agent state history written to append-only Blob with WORM policy
★Model Cards: Documented known limitations of each LLM agent for legal defensibility

The "Never Trust, Always Verify" Principle

LLM agents are powerful but probabilistic. We never let an agent block a bid, award a contract, or delete a record autonomously. Agents recommend. Humans decide. The platform is designed as an augmented intelligence system — the LLM does the reading and reasoning at scale, the human does the judgment and accountability.

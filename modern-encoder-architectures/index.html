The Cast of Characters

One taxonomy, many architectures. Understand the primitive blocks first.

Encoder

Reads input and builds a rich contextual representation. Bidirectional context.

BERT, RoBERTa, DeBERTa

Decoder

Generates output token-by-token. Causal (left-to-right) attention.

GPT, Llama, Claude

Encoder-Decoder

Encoder compresses input; decoder generates output conditioned on it.

T5, BART, Transformer

Bi-Encoder

Two inputs encoded independently into vectors, then scored by similarity.

Sentence-BERT, DPR, OpenAI embeddings

Cross-Encoder

Two inputs concatenated and jointly encoded. More accurate, slower.

MonoT5, Cross-Encoder MS MARCO

MoE + Multimodal

Sparse mixture-of-experts and cross-modal alignment scale capacity.

Mixtral, Flamingo, GPT-4o, Gemini

Self-Attention

Every token attends to every other token. Creates a fully-connected contextual view.

Causal Mask

Decoders mask future tokens. The model can only look backwards — essential for generation.

Cross-Attention

Queries from decoder attend to keys/values from encoder. The bridge between source and target.

Encoder-only

Understand the Input

Bidirectional attention over the full input. Great for classification, embeddings, and “reading” tasks.

  • Every token sees every other token.
  • Outputs contextual token vectors + a pooled [CLS] vector.
  • Cannot generate new tokens on its own.
  • Fine-tuned with a task head on top.

Decoder-only

Generate the Future

Autoregressive models predict the next token using causal self-attention. The dominant LLM design.

  • Each token only attends to previous positions.
  • Trained to maximize likelihood of next token.
  • Strong in-context learning and few-shot prompting.
  • Can be adapted for classification via prompting.
GPT-4LlamaClaudeMistral

Encoder-Decoder: Translate & Transform

Best when input and output are qualitatively different: translation, summarization, structured parsing.

Use When

Input and output vocabularies differ, length changes, or structure must be transformed.

Pros

Clean separation of representation and generation; strong for seq2seq.

Cons

Slower inference; more parameters; eclipsed by large decoder-only LLMs for open-ended tasks.

Bi-Encoder

Encode Once, Compare Forever

Two inputs pass through the same encoder to produce fixed-size vectors. Score = cosine similarity.

  • Each text is encoded independently.
  • Embeddings are precomputed and indexed.
  • Retrieval and clustering are extremely fast.
  • Weaker at capturing fine-grained interactions.
Sentence-BERTDPRContrieverOpenAI embeddings
Cross-Encoder

Joint Understanding

Query and document are concatenated and fed into a single encoder. Every token attends to every other token.

  • Single unified self-attention over both inputs.
  • Captures subtle interactions and token-level alignment.
  • Cannot precompute embeddings: slow at scale.
  • Often used to rerank top-k candidates from a bi-encoder.

Multimodal Architectures

Text, image, audio, and video are encoded into a shared representation space — then fused, aligned, or decoded.

Vision Encoder + Text Decoder

Image patches fed as "tokens" to a transformer. Decoder generates captions or answers.

LLaVA, GPT-4V, Flamingo

Dual Encoders (Contrastive)

Separate encoders for image and text. Trained to align positive pairs in embedding space.

CLIP, ALIGN, SigLIP

Any-to-Any Unified Tokenizer

Images, audio, video tokenized into a discrete vocabulary and processed by one transformer.

GPT-4o, Gemini, Chameleon

Mixture of Experts (MoE)

Scale model capacity without proportionally scaling compute. Only a subset of parameters is active per token.

Why MoE?

Sparse activation keeps FLOPs lower than a dense model with the same parameter count. More knowledge, similar latency.

Router

A learned gating network picks the top-k experts for each token. Load balancing prevents collapse to a single expert.

Trade-offs

Higher memory; complex serving; potential training instability. Works best with expert parallelism.

Pretraining

Build broad representations. Objective depends on architecture: MLM for encoders, CLM for decoders, span corruption for enc-dec.

Alignment

Supervised fine-tuning plus preference optimization makes the model helpful, harmless, and honest.

Distillation

Teacher-student training transfers capability from a large model to a smaller, faster one for edge deployment.

Test-Time Compute

Chain-of-thought, self-consistency, and search multiply effective compute at inference without adding parameters.

Which Architecture Should You Use?

A practical decision guide based on what you are building.

Classification / NER / Sentiment

Encoder-only or a lightweight LLM with a classification head. Fine-tune on labeled examples.

Chat / Code / Creative Writing

Decoder-only LLM with SFT + RLHF. Best in-context learning and generation quality.

Translation / Summarization

Encoder-decoder historically; modern large decoder-only models often match or exceed it.

Semantic Search at Scale

Bi-Encoder for indexing + ANN. Optionally rerank top-k with a Cross-Encoder.

Image Understanding / VQA

Vision encoder + text decoder. For retrieval, use a dual encoder like CLIP.

Max Quality, Long Context, Scale

Large decoder-only with MoE layers and advanced positional encodings (RoPE, ALiBi).

Rule of thumb: Start with a decoder-only LLM if you need generation. Add a bi-encoder if you need fast retrieval. Add a cross-encoder if ranking quality matters more than speed. Use MoE when you need more capacity than a dense model can afford.

Modern Concepts at a Glance

Bite-sized definitions of the ideas that define the current generation of models.

Attention

Weighted aggregation of all positions. Self-attention uses the same sequence; cross-attention uses two sequences.

Positional Encoding

Inject order into permutation-invariant transformers. Modern variants: RoPE, ALiBi, relative positions.

Layer Normalization

Pre-norm is standard today: stabilizes training of very deep transformers.

Residual Connections

Skip connections allow gradients to flow and enable training of hundreds of layers.

Tokenization

BPE, WordPiece, SentencePiece, Unigram. Vocab size and token efficiency affect cost and multilingualism.

Instruction Tuning

Fine-tune on diverse task descriptions to unlock zero-shot generalization.

RLHF / DPO

Align models with human preferences. DPO is simpler and often as effective as PPO-based RLHF.

Quantization

INT8, INT4, GPTQ, AWQ reduce memory and latency with minor quality loss.
